# 05 — Documentation vs. Code Inconsistencies

## Executive Summary

This report consolidates every documentation-vs-code discrepancy that was verified against the actual repository state on 2026-06-15. Each confirmed item below was checked directly against source files (`package.json`, `tsconfig`, compose files, CI workflows, hooks, skill docs, and app source) and is asserted as fact. The dominant theme is **version drift**: top-level docs (`README.md`, `PACKAGE-MODEL.md`, `REFACTORING-LOG.md`) and skills still cite the original `2.0.0` migration-era versions and an 8-package world, while the workspace now has **9 packages** (`@bcl32/data-utils` was added) with versions ranging from `2.1.5` (themes) to `3.1.2` (filters). A second cluster concerns the **publish/changeset pipeline**: docs describe deleted shell scripts (`publish-package.sh`), a non-existent `PackageManager.ts bump`, and a `commit-msg` hook, when the real mechanism is a `.githooks/post-commit` hook that auto-generates changesets plus a single-job CI workflow. A third cluster is the **`moduleResolution: "node"` → `"bundler"`** migration that the TypeScript guide never caught up to. App-level findings are mostly latent code-quality issues (duplicated helpers, leftover `console.log`/debug CSS, hardcoded colors, unused deps).

**Confirmed inconsistencies: 71** (45 doc/skill claims + 26 app claims). **Unverified/refuted claims needing human review: 16.**

> Note: line numbers cited below reflect the prior agent's verification; treat them as approximate anchors. The *reality* and *evidence* columns are the load-bearing content.
